# runtime/core/tag.cpp, runtime/core/tag.h

> [spec:et:def:tag.executorch.runtime.tag]
> enum class Tag : uint32_t

> [spec:et:def:tag.executorch.runtime.tag-to-string-fn]
> size_t tag_to_string(Tag tag, char* buffer, size_t buffer_size)

> [spec:et:sem:tag.executorch.runtime.tag-to-string-fn]
> Writes a string representation of `tag` into `buffer` (capacity `buffer_size`
> including the null terminator) and returns the number of characters that the
> full representation would occupy, excluding the terminator (i.e. `snprintf`
> semantics: the count of bytes that would have been written had the buffer been
> large enough).
>
> Behavior depends on the compile-time flag `ET_ENABLE_ENUM_STRINGS` (on by
> default):
> - If enabled: selects a name string by switching on `tag`. Each enumerator of
>   `Tag` maps to its own identifier text — the twelve tags in declaration order
>   are "None", "Tensor", "String", "Double", "Int", "Bool", "ListBool",
>   "ListDouble", "ListInt", "ListTensor", "ListScalar", "ListOptionalTensor"
>   (the set generated by EXECUTORCH_FORALL_TAGS). Any value not matching a known
>   tag maps to "Unknown". Then formats that name via `snprintf(buffer,
>   buffer_size, "%s", name_str)`.
> - If disabled: formats the tag's underlying integer value via
>   `snprintf(buffer, buffer_size, "%d", static_cast<int>(tag))`, i.e. the
>   decimal index of the enumerator (0 for None, 1 for Tensor, ... following
>   declaration order).
>
> In both branches: `snprintf` null-terminates `buffer` and truncates the copied
> content to fit `buffer_size` when the full string is longer, but the returned
> length is always the untruncated length. `kTagNameBufferSize` (19) is the
> buffer size that fits the longest tag name ("ListOptionalTensor", 18 chars)
> plus the terminator. This overload is distinct from the header-inline
> `tag_to_string(Tag)` which returns a static `const char*` without a buffer.
