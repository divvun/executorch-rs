> [spec:et:sem:op-sinh.torch.executor.native.sinh-out-fn]
> Computes the hyperbolic sine of `in` element-wise into `out`. Generated by
> `DEFINE_UNARY_UFUNC_REALHBBF16_TO_FLOATHBF16(sinh_out, std::sinh)`, so it
> delegates to the pattern `[spec:et:sem:pattern.torch.executor.native.internal.unary-ufunc-realhbbf16-to-floathbf16-fn]`
> with `fn_float = fn_double = std::sinh`. Concretely:
>
> - `out` must be a floating-point tensor (Half, Float, Double, or BFloat16);
>   ET_KERNEL_CHECK sets Error::InvalidArgument and returns `out` unchanged
>   otherwise.
> - Resizes `out` to `in.sizes()`; on failure sets Error::InvalidArgument and
>   returns `out`.
> - Requires `in` and `out` to have the same dim order; else Error::InvalidArgument.
> - Accepts input dtype from REALHBBF16 = {Byte, Char, Short, Int, Long, Bool,
>   Half, Float, Double, BFloat16}; output dtype from FLOATHBF16 = {Half, Float,
>   Double, BFloat16}.
> - Applies `sinh` element-wise over all `in.numel()` elements in flat index
>   order: for a Double input each value is computed as `std::sinh` in double;
>   for every other input dtype the value is cast to float, `std::sinh` is
>   applied in float, then the result is cast to the output ctype. NaN input
>   yields NaN; +/-inf yields +/-inf per the underlying `std::sinh`; large
>   magnitudes may overflow to +/-inf.
> - Returns `out`.
