# kernels/portable/cpu/op_reciprocal.cpp

> [spec:et:def:op-reciprocal.torch.executor.native.reciprocal-fn]
> T reciprocal(T x)

> [spec:et:sem:op-reciprocal.torch.executor.native.reciprocal-fn]
> Scalar elementwise kernel. For input value `x` of floating type `T`
> (instantiated as `float` and `double`), computes `1.0 / x` and returns it as
> `T`. The literal `1.0` is a `double`, so the division is performed in `double`
> precision and the result is narrowed back to `T` on return. Follows IEEE-754
> semantics of the platform: `reciprocal(0) == +inf`, `reciprocal(-0) == -inf`,
> `reciprocal(+inf) == 0`, and `reciprocal(NaN) == NaN`. No rounding beyond the
> `double`→`T` narrowing on return.

> [spec:et:def:op-reciprocal.torch.executor.native.reciprocal-out-fn]
> Tensor& reciprocal_out(KernelRuntimeContext& ctx, const Tensor& in, Tensor& out) [generated by DEFINE_UNARY_UFUNC_REALHBBF16_TO_FLOATHBF16]

> [spec:et:sem:op-reciprocal.torch.executor.native.reciprocal-out-fn]
> Generated by `DEFINE_UNARY_UFUNC_REALHBBF16_TO_FLOATHBF16(reciprocal_out,
> reciprocal)`, which delegates to the shared pattern
> `internal::unary_ufunc_realhbbf16_to_floathbf16(reciprocal, reciprocal, ctx,
> in, out)` (see `[spec:et:sem:unary-ufunc-realhbbf16-to-floathbf16.torch.executor.native.internal.unary-ufunc-realhbbf16-to-floathbf16-fn]`).
> Concretely:
>
> - ET_KERNEL_CHECK: `out` must be a floating-point type (`tensor_is_floating_type`,
>   i.e. one of Half, Float, Double, BFloat16). On failure set
>   `Error::InvalidArgument` on the context and return `out` unchanged.
> - Resize `out` to `in.sizes()` via `resize_tensor`; on failure set
>   `Error::InvalidArgument` (message "Failed to resize output tensor.") and
>   return `out`.
> - ET_KERNEL_CHECK: `in` and `out` must have the same dim order
>   (`tensors_have_same_dim_order`); else `Error::InvalidArgument`, return `out`.
> - Accepted input dtypes are REALHBBF16 = {Byte, Char, Short, Int, Long, Half,
>   Float, Double, Bool, BFloat16}. Accepted output dtypes are FLOATHBF16 =
>   {Half, Float, Double, BFloat16}. Input and output dtypes need not match.
> - Elementwise over `in.numel()` elements in flat memory order (contiguous per
>   the shared dim order): read `val_in` as the input ctype. If the input ctype
>   is `double`, compute in `double` (`reciprocal(static_cast<double>(val_in))`);
>   otherwise widen to `float` and compute in `float`
>   (`reciprocal(static_cast<float>(val_in))`). Cast the scalar result to the
>   output ctype and store it at the same flat position. Integer/bool inputs are
>   thus widened to float before the reciprocal is taken.
> - NaN/inf follow the scalar rule in
>   `[spec:et:sem:op-reciprocal.torch.executor.native.reciprocal-fn]`.
> - Returns `out`.
