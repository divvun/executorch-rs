> [spec:et:sem:op-sqrt.torch.executor.native.sqrt-out-fn]
> Computes the square root of `in` element-wise into `out`. Generated by
> `DEFINE_UNARY_UFUNC_REALHBBF16_TO_FLOATHBF16(sqrt_out, std::sqrt)`, delegating
> to `[spec:et:sem:pattern.torch.executor.native.internal.unary-ufunc-realhbbf16-to-floathbf16-fn]`
> with `fn_float = fn_double = std::sqrt`. Concretely:
>
> - `out` must be a floating-point tensor (Half, Float, Double, BFloat16);
>   ET_KERNEL_CHECK sets Error::InvalidArgument and returns `out` unchanged
>   otherwise.
> - Resizes `out` to `in.sizes()`; on failure Error::InvalidArgument, returns `out`.
> - Requires `in` and `out` to have the same dim order; else Error::InvalidArgument.
> - Input dtype from REALHBBF16 = {Byte, Char, Short, Int, Long, Bool, Half,
>   Float, Double, BFloat16}; output dtype from FLOATHBF16 = {Half, Float,
>   Double, BFloat16}.
> - Applies `sqrt` element-wise over all `in.numel()` elements in flat order:
>   Double input computed in double via `std::sqrt`; any other input dtype cast
>   to float, `std::sqrt` in float, then cast to the output ctype. Negative
>   inputs yield NaN; NaN yields NaN; +inf yields +inf per `std::sqrt`.
> - Returns `out`.
