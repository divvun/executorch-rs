> Generated by `DEFINE_UNARY_UFUNC_REALHBBF16_TO_FLOATHBF16(atan_out, std::atan)`,

>
> Concretely: validates `out` is a floating type (`tensor_is_floating_type` →
> `InvalidArgument`), resizes `out` to `in.sizes()` ("Failed to resize output
> tensor." → `InvalidArgument`), checks `tensors_have_same_dim_order(in, out)`
> (→ `InvalidArgument`). Accepted input dtypes (`ET_SWITCH_REALHBBF16_TYPES`):
> {Byte, Char, Short, Int, Long, Half, Float, Double, Bool, BFloat16}; output
> dtypes (`ET_SWITCH_FLOATHBF16_TYPES`): {Half, Float, Double, BFloat16}.
> Element-wise over `in.numel()` flat elements: if CTYPE_IN is `double`, compute
> `std::atan((double)val_in)`; otherwise `std::atan((float)val_in)`; cast to
> CTYPE_OUT. IEEE `std::atan` semantics: domain all reals, range `(-pi/2, pi/2)`;
> NaN yields NaN; `+/-inf` yields `+/-pi/2`. Returns `out`.
