> [spec:et:sem:op-ceil.torch.executor.native.ceil-out-fn]
> Generated by `DEFINE_UNARY_UFUNC_REALHBF16(ceil_out, std::ceil)`, which
> forwards to `internal::unary_ufunc_realhbf16(std::ceil, std::ceil, ctx, in,
> out)` — see
> `[spec:et:sem:unary-ufunc-realhbf16.torch.executor.native.internal.unary-ufunc-realhbf16-fn]`.
> Unlike the *_to_floathbf16 variant, this pattern requires the output to have
> the same shape and dtype as the input.
>
> Concrete behavior:
> 1. Resize `out` to `in.sizes()`; if resize does not return Error::Ok set
>    Error::InvalidArgument (message "Failed to resize output tensor.") and
>    return `out` unchanged.
> 2. ET_KERNEL_CHECK: `tensors_have_same_shape_and_dtype(in, out)`; on failure
>    set Error::InvalidArgument and return `out` unchanged.
> 3. ET_KERNEL_CHECK: `tensors_have_same_dim_order(in, out)`; on failure set
>    Error::InvalidArgument and return `out` unchanged.
> 4. Dispatch on `in.scalar_type()` over REALHBF16 = {Byte, Char, Short, Int,
>    Long, Half, Float, Double, BFloat16}; other dtypes set Error::InvalidArgument
>    and return `out` unchanged.
> 5. Elementwise over `in.numel()` elements in flat order: if the C-type is
>    `double`, compute `std::ceil` in double; otherwise cast the input to `float`,
>    compute `std::ceil` in float, and cast the result back to the C-type. Store
>    to `out`. (For integer input types ceil is a no-op numerically; the value
>    round-trips.) std::ceil semantics: smallest integer value not less than x;
>    ceil(+/-0)=+/-0, ceil(+/-inf)=+/-inf, ceil(NaN)=NaN.
> 6. Return `out`.
