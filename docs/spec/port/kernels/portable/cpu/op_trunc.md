# kernels/portable/cpu/op_trunc.cpp

> [spec:et:def:op-trunc.torch.executor.native.trunc-out-fn]
> Tensor& trunc_out(KernelRuntimeContext& ctx, const Tensor& in, Tensor& out) [generated by DEFINE_UNARY_UFUNC_REALHBF16]

> [spec:et:sem:op-trunc.torch.executor.native.trunc-out-fn]
> Truncates each element of `in` toward zero into `out`. Generated by
> `DEFINE_UNARY_UFUNC_REALHBF16(trunc_out, std::trunc)`, delegating to
> `[spec:et:sem:pattern.torch.executor.native.internal.unary-ufunc-realhbf16-fn]`
> with `fn_float = fn_double = std::trunc`. Note this pattern differs from the
> `to_floathbf16` variant: input and output must have the SAME dtype and shape.
> Concretely:
>
> - Resizes `out` to `in.sizes()`; on failure ET_KERNEL_CHECK_MSG sets
>   Error::InvalidArgument ("Failed to resize output tensor.") and returns `out`.
> - Requires `in` and `out` to have the same shape AND dtype
>   (tensors_have_same_shape_and_dtype); else Error::InvalidArgument, returns `out`.
> - Requires `in` and `out` to have the same dim order; else Error::InvalidArgument.
> - Dtype from REALHBF16 = {Byte, Char, Short, Int, Long, Half, Float, Double,
>   BFloat16} (note: NO Bool). The single switched CTYPE is used for both input
>   and output.
> - Applies `trunc` element-wise over all `in.numel()` elements in flat order:
>   Double computed in double via `std::trunc`; every other dtype cast to float,
>   `std::trunc` in float, then cast back to CTYPE. For integer CTYPEs the value
>   is already integral so `std::trunc` returns it unchanged. NaN yields NaN;
>   +/-inf yields +/-inf per `std::trunc`.
> - Returns `out`.
