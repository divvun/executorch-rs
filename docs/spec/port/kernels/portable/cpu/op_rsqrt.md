# kernels/portable/cpu/op_rsqrt.cpp

> [spec:et:def:op-rsqrt.torch.executor.native.rsqrt-fn]
> T rsqrt(T x)

> [spec:et:sem:op-rsqrt.torch.executor.native.rsqrt-fn]
> Scalar elementwise kernel. For input value `x` of floating type `T`
> (instantiated as `float` and `double`), computes `1.0 / std::sqrt(x)` and
> returns it as `T`. `std::sqrt` is evaluated at type `T`; the `1.0` numerator
> is a `double`, so the division is performed in `double` and narrowed back to
> `T` on return. IEEE-754 semantics: `rsqrt(0) == +inf`, `rsqrt(x<0) == NaN`
> (sqrt of a negative is NaN), `rsqrt(+inf) == 0`, `rsqrt(NaN) == NaN`.

> [spec:et:def:op-rsqrt.torch.executor.native.rsqrt-out-fn]
> Tensor& rsqrt_out(KernelRuntimeContext& ctx, const Tensor& in, Tensor& out) [generated by DEFINE_UNARY_UFUNC_REALHBBF16_TO_FLOATHBF16]

> [spec:et:sem:op-rsqrt.torch.executor.native.rsqrt-out-fn]
> Generated by `DEFINE_UNARY_UFUNC_REALHBBF16_TO_FLOATHBF16(rsqrt_out, rsqrt)`,
> delegating to `internal::unary_ufunc_realhbbf16_to_floathbf16(rsqrt, rsqrt,
> ctx, in, out)` (see
> `[spec:et:sem:unary-ufunc-realhbbf16-to-floathbf16.torch.executor.native.internal.unary-ufunc-realhbbf16-to-floathbf16-fn]`).
> Behavior is identical to
> `[spec:et:sem:op-reciprocal.torch.executor.native.reciprocal-out-fn]` except
> the per-element scalar function is `rsqrt`
> (`[spec:et:sem:op-rsqrt.torch.executor.native.rsqrt-fn]`) instead of
> reciprocal:
>
> - ET_KERNEL_CHECK: `out` must be floating type; else `Error::InvalidArgument`,
>   return `out`.
> - Resize `out` to `in.sizes()`; on failure `Error::InvalidArgument` (message
>   "Failed to resize output tensor."), return `out`.
> - ET_KERNEL_CHECK: same dim order for `in`/`out`; else `Error::InvalidArgument`.
> - Input dtypes REALHBBF16, output dtypes FLOATHBF16 (need not match).
> - Elementwise over `in.numel()`: read `val_in`; compute in `double` if input
>   ctype is `double`, else in `float`; store `rsqrt(val_in)` cast to the output
>   ctype. Integer/bool inputs are widened to float first.
> - Returns `out`.
