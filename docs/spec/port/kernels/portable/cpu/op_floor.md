> [spec:et:sem:op-floor.torch.executor.native.floor-out-fn]
> Generated by `DEFINE_UNARY_UFUNC_REALHBF16(floor_out, std::floor)`; delegates to
> `internal::unary_ufunc_realhbf16(std::floor, std::floor, ctx, in, out)`, whose behavior is
> `[spec:et:sem:unary-ufunc-realhbf16.torch.executor.native.internal.unary-ufunc-realhbf16-fn]`.
> Unlike the REALHBBF16_TO_FLOATHBF16 pattern, this pattern requires `in` and `out` to have the
> SAME dtype. Every failure path sets the error on `ctx` and returns `out` unchanged.
> 1. ET_KERNEL_CHECK `resize_tensor(out, in.sizes()) == Error::Ok`, else InvalidArgument.
> 2. ET_KERNEL_CHECK `tensors_have_same_shape_and_dtype(in, out)`, else InvalidArgument.
> 3. ET_KERNEL_CHECK `tensors_have_same_dim_order(in, out)`, else InvalidArgument.
> 4. Dispatch over `in.scalar_type()` restricted to REALHBF16
>    (`{Byte, Char, Short, Int, Long, Half, Float, Double, BFloat16}` — note: NO Bool). Elementwise
>    in flat index order over `in.numel()` elements: if the dtype is `double`, compute
>    `std::floor(static_cast<double>(x))`; otherwise compute
>    `static_cast<CTYPE>(std::floor(static_cast<float>(x)))`. Integer dtypes are floored trivially
>    (value unchanged since std::floor of an integral value is itself). NaN/inf follow `std::floor`
>    (floor(±inf)=±inf, floor(NaN)=NaN).
> 5. Return `out`.
