> [spec:et:sem:op-tanh.torch.executor.native.tanh-out-fn]
> Computes the hyperbolic tangent of `in` element-wise into `out`. Generated by
> `DEFINE_UNARY_UFUNC_REALHBBF16_TO_FLOATHBF16(tanh_out, std::tanh)`, delegating
> to `[spec:et:sem:pattern.torch.executor.native.internal.unary-ufunc-realhbbf16-to-floathbf16-fn]`
> with `fn_float = fn_double = std::tanh`. Concretely:
>
> - `out` must be a floating-point tensor (Half, Float, Double, BFloat16); else
>   ET_KERNEL_CHECK sets Error::InvalidArgument and returns `out` unchanged.
> - Resizes `out` to `in.sizes()`; on failure Error::InvalidArgument, returns `out`.
> - Requires `in` and `out` to have the same dim order; else Error::InvalidArgument.
> - Input dtype from REALHBBF16 = {Byte, Char, Short, Int, Long, Bool, Half,
>   Float, Double, BFloat16}; output dtype from FLOATHBF16 = {Half, Float,
>   Double, BFloat16}.
> - Applies `tanh` element-wise over all `in.numel()` elements in flat order:
>   Double input in double via `std::tanh`; any other input dtype cast to float,
>   `std::tanh` in float, then cast to output ctype. NaN yields NaN; +/-inf
>   saturate to +/-1 per `std::tanh`.
> - Returns `out`.
