> [spec:et:sem:op-sin.torch.executor.native.sin-out-fn]
> Generated by `DEFINE_UNARY_UFUNC_REALHBBF16_TO_FLOATHBF16(sin_out, std::sin)`,
> delegating to `internal::unary_ufunc_realhbbf16_to_floathbf16(std::sin,
> std::sin, ctx, in, out)` (see
> `[spec:et:sem:unary-ufunc-realhbbf16-to-floathbf16.torch.executor.native.internal.unary-ufunc-realhbbf16-to-floathbf16-fn]`).
> Behavior is identical to
> `[spec:et:sem:op-reciprocal.torch.executor.native.reciprocal-out-fn]` except
> the per-element scalar function is `std::sin`:
>
> - ET_KERNEL_CHECK: `out` must be floating type; else `Error::InvalidArgument`,
>   return `out`.
> - Resize `out` to `in.sizes()`; on failure `Error::InvalidArgument` (message
>   "Failed to resize output tensor."), return `out`.
> - ET_KERNEL_CHECK: same dim order for `in`/`out`; else `Error::InvalidArgument`.
> - Input dtypes REALHBBF16 = {Byte, Char, Short, Int, Long, Half, Float, Double,
>   Bool, BFloat16}; output dtypes FLOATHBF16 = {Half, Float, Double, BFloat16}
>   (need not match).
> - Elementwise over `in.numel()`: read `val_in`; if input ctype is `double`
>   compute `std::sin` in `double`, else widen to `float` and compute `std::sin`
>   in `float`; store the result cast to the output ctype. Integer/bool inputs
>   are widened to float first.
> - `std::sin` semantics: `sin(±0) == ±0`, `sin(±inf) == NaN`, `sin(NaN) ==
>   NaN`; the mathematical sine otherwise.
> - Returns `out`.
